# 加密货币投资记录 (Crypto Trade Tracker) - 用例说明

## 参与者 (Actors)
* **交易者 (Trader)**：系统的唯一用户，负责输入交易数据并查看盈亏统计。

## 核心用例 (Use Cases)

### UC1: 记录基础交易单 (Record Transaction)
* **简述**：交易者在交易所发出买入或卖出动作后，将单笔成交记录录入系统。同时可在此步骤直接关联波段。
* **前置条件**：无。
* **基本流程**：
  1. 交易者选择“新增交易”。
  2. 填写交易详情：
     * 交易时间 (Date/Time)
     * 交易对/币种 (e.g., BTC/USDT)
     * 交易方向 (买入 Buy / 卖出 Sell)
     * 成交单价 (Price)
     * 成交数量 (Amount)
     * 手续费 (Fee)
  3. **选择波段关联（可选）**：
     * 交易者可以选择将该笔交易**关联到一个或多个现有的“交易波段”**。
     * 或者选择**“新建一个交易波段”**并立即关联。
     * *注：一笔交易单支持同时关联到多个不同的交易波段中（多对多关系），方便用户将一次大额买入拆分用于不同策略的波段。*
  4. 系统保存该单笔交易及其波段关联关系。
* **扩展流程**：提供从主流交易所（如 Binance/OKX）导出的 CSV 文件批量导入功能（作为一个进阶功能规划）。

### UC2: 创建与管理交易波段 (Manage Trade Position) - 【核心】
* **简述**：将一次或多次基础交易单关联起来，形成一个完整的“开仓到平仓”的波段，用于计算该次操作波段的真实盈亏。
* **前置条件**：系统内已有相关的“记录基础交易单”，或者在创建波段时同步创建单笔交易。
* **基本流程**：
  1. 交易者选择“新建波段”（或在录入交易时触发新建）。
  2. 选择交易对（例如 SOL/USDT）以及波段名称/标签。
  3. 选取系统中已有的或新录入的“买入”记录作为**开仓**。（支持分批买入，同一笔买入可被多个波段共用）
  4. 随着行情发展，选取/录入相应的“卖出”记录作为**平仓**。（支持分批卖出）
  5. 只要平仓数量累计达到开仓数量，该波段即视为闭环。
  6. 系统自动计算该波段的：
     * 综合开仓均价、综合平仓均价
     * 已实现盈亏 (Realized PnL，含扣除手续费后的净利润)
     * 投资回报率 (ROI)
* **波段状态流转**：
  * **进行中 (Open)**：有开仓记录，但尚未完全平仓。此时系统根据当前市场标价推算【浮动盈亏】。
  * **已闭环 (Closed)**：关联的卖出数量已涵盖所需平仓部分。固化该波段的【实际盈亏】。

### UC3: 计算交易波段盈亏与关键指标 (Calculate Position PnL & Metrics)
* **简述**：系统根据波段内关联的买入和卖出明细，自动结算出各项核心财务指标，为交易者呈现真实的盈亏状况。
* **前置条件**：该交易波段下至少关联了一笔买入或卖出记录。
* **核心指标计算逻辑**：
  1. **开仓总成本 (Total Cost)** = Σ (单笔买入单价 × 对应买入数量)
  2. **综合开仓均价 (Average Entry Price)** = 开仓总成本 / 总买入数量
  3. **平仓总收入 (Total Revenue)** = Σ (单笔卖出单价 × 对应卖出数量)
  4. **综合平仓均价 (Average Exit Price)** = 平仓总收入 / 总卖出数量
  5. **总手续费支出 (Total Fees)** = 波段内所有关联交易单（含买与卖）被分摊计入该波段的手续费总计。
  6. **已实现盈亏 (Realized PnL)** = 平仓总收入 - (综合开仓均价 × 累计已卖出数量) - 已平仓部分对应的手续费（或视用户需求扣除所有已生手续费）
  7. **浮动盈亏 / 未实现盈亏 (Unrealized PnL)** = (当前市场实时价格 - 综合开仓均价) × 剩余持仓数量
     * *注：通常仅在波段处于“进行中(Open)”状态，且有未平仓余额时计算。*
  8. **总收益率 (ROI, Return on Investment)** = (已实现盈亏 + 浮动盈亏) / 开仓总成本 × 100%
* **触发机制**：
  * 当用户新增/删除/修改系统内的波段关联交易时，数据状态即刻重新计算。
  * 若波段包含“当前持仓”且系统接入了实时现货标价（或用户手动刷新价格），则浮动盈亏随价格波动实时更新。

### UC4: 记录交易复盘与笔记 (Trade Journaling)
* **简述**：针对特定的“交易波段”，交易者可以记录自己的操作逻辑和反思。
* **前置条件**：存在一个“交易波段”（进行中或已闭环）。
* **基本流程**：
  1. 交易者进入某一个“交易波段”的详情页。
  2. 填写/修改复盘信息：
     * 开仓理由（如：突破阻力位、宏观利好消息、定投策略）
     * 平仓理由（如：触发止盈位/止损位、恐慌抛售、基本面改变）
     * 情绪评分与反思总结
  3. 系统保存笔记，并在历史波段回顾中展示，帮助交易者优化交易系统。

### UC5: 查看总览看板 (View Dashboard)
* **简述**：交易者查看自身所有交易波段的宏观表现数据。
* **前置条件**：系统内存在已闭环的交易波段及进行中的波段。
* **基本流程**：
  1. 交易者进入首页看板。
  2. 系统展示统计指标：
     * **累计利润 (Total PnL)**：所有已闭环波段的净盈亏总和。
     * **胜率 (Win Rate)**：盈利波段数 / 总闭环波段数。
     * **盈亏比 (Risk/Reward Ratio)**：平均单次盈利金额 / 平均单次亏损金额。
     * **当前持仓 (Open Positions)**：目前仍处于“进行中”状态的波段列表及其浮动盈亏情况。

## 待定与边界说明
* **交易性质**：当前设计主要针对**现货交易（Spot Trading）**，即“低买高卖”逻辑。如果后续需要支持合约（如做空机制），对应的开平仓逻辑（买入平空、卖出开空）需要进一步扩展。
* **实时币价**：系统初期计算“进行中(Open)”波段的浮动盈亏，可以采用调用公共免费 API（如 CoinGecko 或 Binance 免费接口）获取当前标价，或者暂时提供价格手动刷新按钮。
